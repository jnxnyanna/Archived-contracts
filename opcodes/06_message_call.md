#### **6. Message Call Operations**
###### Opcode message call pada Ethereum Virtual Machine digunakan untuk melakukan interaksi antar kontrak di blockchain Ethereum. Opcode ini memungkinkan kontrak memanggil fungsi kontrak lain, menjalankan kode kontrak lain dalam konteks tertentu, atau melakukan pemanggilan read-only tanpa mengubah state. Opcode utama dalam kategori ini meliputi `CALL` (`0xf1`), `DELEGATECALL` (`0xf4`), dan `STATICCALL` (`0xfa`). Operasi ini bekerja dengan mekanisme message call, yaitu membuat eksekusi baru dengan parameter seperti alamat tujuan, gas, calldata, serta memory offset untuk input dan output data.
 
 No | Opcode                   | Sintaks                                                              | Kategori     | Deskripsi                                                                                                                                                                               | Contoh                                   | Gas 
 -- | ------------------------ | -------------------------------------------------------------------- | ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------- |-----
 **1.** | `CALL` _(Hex: `0xf1`)_         | `call(gas, address, value, argsOffset, argsSize, returnOffset, returnSize)`  | Message Call | Melakukan pemanggilan kontrak lain dengan mengirim ETH dan data. Eksekusi terjadi pada konteks kontrak tujuan (storage milik kontrak tujuan).                                           | `call(10000, target, 1 ether, 0, 32, 0, 32)` | _Dinamis_
 **2.** | `DELEGATECALL` _(Hex: `0xf4`)_ | `delegatecall(gas, address, argsOffset, argsSize, returnOffset, returnSize)` | Message Call | Menjalankan kode kontrak lain tetapi tetap menggunakan storage, `msg.sender`, dan `msg.value` milik kontrak pemanggil. Umumnya digunakan pada pola proxy contract dan upgradeable contract. | `delegatecall(10000, implementation, 0, 32, 0, 32)`  |_Dinamis_
 **3.** | `STATICCALL` _(Hex: `0xfa`)_   | `staticcall(gas, address, argsOffset, argsSize, returnOffset, returnSize)`   | Message Call | Melakukan pemanggilan kontrak lain dalam mode read-only. Opcode ini melarang perubahan state seperti `SSTORE`, `CREATE`, atau `SELFDESTRUCT` selama eksekusi.                                 | `staticcall(10000, target, 0, 32, 0, 32)`  |_Dinamis_
 **4.** | `CALLCODE` _(Hex: `0xf2`)_|`callcode(gas, address, value, argsOffset, argsSize, returnOffset, returnSize)`|Message Call|⚠️ **DEPRECATED!** Menjalankan kode dari kontrak lain tetapi menggunakan storage milik kontrak pemanggil. Dalam konteks ini, `msg.sender` tetap menjadi kontrak pemanggil dan `msg.value` dapat diteruskan. Opcode ini awalnya digunakan untuk pola library contract, namun kini jarang digunakan karena telah digantikan oleh `DELEGATECALL` yang mempertahankan konteks `msg.sender` dan `msg.value` dari pemanggilan asli.|`callcode(10000, target, 1 ether, 0, 32, 0, 32)`|_Dinamis_
